Polish 怎么解

把任何已部署的页面变成可编辑画布。设计师拖到位 → 一键导出 AI-ready 改动包 → 研发或 AI 直接落代码。

1

打开页面

任何 URL：Vercel preview、Storybook、自己服务器

2

点 bookmarklet

书签一点 → 底部出现 Polish 工具栏

3

拖 / 改 / 备注

真 CSS 编辑：颜色、字号、间距、圆角

4

导出 .md

粘给 Cursor / Claude → 代码改了

核心特性

不是 Chrome DevTools 的换皮。每一个特性都为了 design-to-code 的最后一公里。

真 CSS 编辑

颜色 / 字号 / 字重 / 行高 / 间距 / 圆角 / 阴影 / 透明度。改 padding 让兄弟元素正确 reflow，不是 transform 假象。

AI-native 导出

导出 markdown 含选择器 + CSS diff + 备注。Cursor / Claude 一眼能消化，不需要研发翻译。

元素级备注

"这里再透气一点"这种意图也能传递。LLM 读到备注会更理解设计语义。

分享链接

改动编码到 URL，研发打开同样链接看到所有覆盖。不用聊天框传截图传文字。

响应式预览

一键切 375 / 768 / 1024 / 1280 / 1440 视口验证多断点。

Figma 参考图覆盖

上传设计稿半透明叠到当前页面，逐像素对齐。

Tailwind 感知

检测项目用 Tailwind 时，导出会建议等价 utility class（pl-6 而不是 padding-left:24px）。

零依赖 · 不污染生产

单文件、无 npm 树，bookmarklet 也能用。没有 ?polish=1 时完全 no-op。

30 秒开始

三种方式，挑一种。

A

Bookmarklet（推荐设计师 / PM）

把下面这颗按钮拖到浏览器书签栏，然后在任何页面点一下书签。

📐 拖我到书签栏 → Polish
B

一行 script（推荐研发）

在项目 html 末尾加这行，然后 URL 带 ?polish=1 启用。

<script src="https://polish.bowie.top/polish.js"></script>
C

npm（v0.1 即将发布）

npm install polish
import 'polish';
